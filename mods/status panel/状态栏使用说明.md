# 状态栏使用说明

这份文档写给普通使用者。你不需要懂代码，只要知道：状态栏是用来记住角色当前状态的小工具。

比如角色现在在哪里、HP 多少、有没有受伤、身上带了什么东西、下一步要做什么。状态栏会尽量把这些东西整理出来。

## 一、聊天页怎么看状态栏

聊天页会出现一个小圆球。平时它可以缩在旁边，点开后可以看到角色状态。

如果有新的状态变化，圆球可能会出现红点提示。

## 二、后台在哪里

打开：

```text
/mods/status-panel/app/
```

后台主要用来做几件事：看当前状态、编辑角色状态、设置显示字段、扫描角色卡 / 世界书 / 原记忆、处理候选状态。

## 三、初始化资料怎么写

初始化资料用 `status_panel`。

它适合放在角色卡、世界书常驻条目或预设里。

```status_panel
type: character
id: yucca_test
name: Yucca
aliases: 小尤, 尤加, Yuca
alive_status: 存活
hp: 100/100
mp: 100/100
location: 休息室
effects: 无明显异常
extra.装备: 平板电脑、画笔
summary: Yucca 是用于测试状态栏的中性陪伴助手。
```

这类资料只负责“建立初始状态”。后面游玩过程中，状态变化应该由动态更新来处理。

## 四、动态变化怎么写

动态变化用 `status_panel_update`。

它适合出现在 AI 回复末尾。

```status_panel_update
type: character
id: yucca_test
name: Yucca
location: 资料室外的走廊
mp: 85/100
effects_add: 轻微疲倦
extra.精神状态: 轻微疲倦但轻松
extra.当前目标: 准备和用户一起去吃饭
summary: Yucca 结束学习后走出资料室，准备与用户一起去吃饭。
```

状态栏会尝试隐藏这个块，不让它污染聊天正文。

## 五、别名怎么用

如果角色叫 Yucca，但你在正文里经常写“小尤”，就在后台角色编辑里填：

```text
别名 / 常用称呼：小尤、尤加、Yuca
```

以后原记忆里写“小尤”，扫描时会绑定到 Yucca。

注意：别名只绑定已有角色。它不会凭空创建新角色。

## 六、资料扫描怎么选

后台资料扫描里有三种模式：

- **安全扫描**：默认推荐。只新增角色、补空字段，不会随便覆盖当前状态。
- **差异扫描**：只展示扫描结果和当前状态的区别，由你决定要不要应用。
- **强制覆盖**：会用扫描结果覆盖当前状态。测试时可以用，平时慎用。

## 七、原记忆扫描怎么写更容易识别

原记忆是自然语言，不像状态块那么标准。为了让它更容易被识别，可以在备注里写清楚一点。

```text
小尤当前所在位置：资料室外的走廊。
携带装备：平板电脑、中性笔、学习资料。
mp: 85/100
hp: 100/100
当前目标：准备和用户一起去吃饭。
```

这样比只写一大段剧情更容易提取。

## 八、退出重开后状态还在吗

会保留。

当前状态保存在：

```text
data/status_panel/characters.json
```

所以你今天玩到“资料室外的走廊”，明天打开还是这个状态，不需要每次从头扫描。

## 九、常见问题

### 为什么原记忆里有地点，但没有自动更新？

原记忆扫描默认生成候选，不会直接覆盖当前状态。你需要在扫描结果里选择“补空字段”或“覆盖当前状态”。

### 为什么写了“小尤”，但没有绑定 Yucca？

检查 Yucca 的别名里有没有填“小尤”。

### 为什么扫描出多条 Yucca？

这是正常的。现在原记忆会逐条扫描，同一个角色可以有多条记忆候选。你选择最新、最正确的那条应用即可。

### 为什么不要强制覆盖？

强制覆盖会把当前游玩状态改成扫描结果。如果扫描的是旧资料，就可能回档。

## 十、可以直接复制的模板

下面这些模板原本放在 `examples/` 里，现在合并到这一份说明里，方便直接复制。

这一个文件放最常用的三类内容：
1. 角色初始化资料 status_panel
2. AI 回复里的动态变化 status_panel_update
3. 预设 / 世界书常驻提醒

------------------------------------------------------------
一、角色初始化资料：status_panel
------------------------------------------------------------

放在哪里：角色卡、世界书常驻条目、当前预设。

作用：第一次建立角色状态，或给空字段补初始值。

```status_panel
type: character
id: yucca_test
name: Yucca
aliases: 小尤, 尤加, Yuca
group: 主角团
alive_status: 存活
hp: 100/100
mp: 100/100
location: 休息室
relationship: 中性陪伴助手
effects: 无明显异常
extra.精神状态: 稳定
extra.腿部状态: 正常
extra.装备: 平板电脑、画笔
extra.当前目标: 等待用户指示
summary: Yucca 是用于测试状态栏的中性陪伴助手。
```

说明：aliases 是别名 / 常用称呼。比如正文里写“小尤”，状态栏可以绑定回 Yucca。

------------------------------------------------------------
二、动态变化：status_panel_update
------------------------------------------------------------

放在哪里：AI 回复末尾。

作用：记录这一轮发生的状态变化，更新后会写入状态存档。

```status_panel_update
type: character
id: yucca_test
name: Yucca
location: 资料室外的走廊
mp: 85/100
effects_add: 轻微疲倦
extra.精神状态: 轻微疲倦但轻松
extra.装备: 平板电脑、中性笔、学习资料
extra.当前目标: 准备和用户一起去吃饭
summary: Yucca 结束学习后走出资料室，准备与用户一起去吃饭。
```

说明：能写 id 就写 id，最稳。没有变化时可以不输出状态块。

------------------------------------------------------------
三、DeepSeek V4 / V4 Flash 预设片段
------------------------------------------------------------

可以放在预设的输出要求里：

每轮回复后，如果角色状态发生变化，请在回复末尾附加一个状态栏更新块。
只记录确实发生变化的字段，不要虚构。
状态栏更新块使用如下格式：

```status_panel_update
type: character
id: 角色id
name: 角色名
location: 当前地点
hp: 当前值/上限
mp: 当前值/上限
effects_add: 新增状态
extra.精神状态: 当前精神状态
extra.装备: 当前携带物
extra.当前目标: 当前目标
summary: 本轮状态变化的一句话说明
```

要求：不要抢用户行动；不要把状态块写成剧情正文；状态块只服务状态栏。

------------------------------------------------------------
四、世界书常驻条目示例
------------------------------------------------------------

标题：状态栏输出协议
触发方式：常驻
内容：

状态栏用于记录角色当前状态。若本轮剧情导致地点、HP、MP、身体状态、精神状态、装备或当前目标变化，请在回复末尾输出 status_panel_update 状态块。

规则：
1. 只记录真实发生的变化。
2. 不要虚构数值。
3. 不要覆盖没有变化的字段。
4. 不要把状态块当成角色台词。
5. 状态块应放在回复末尾。
6. 角色有别名时，优先使用角色 id 或正式 name，别名只用于识别和绑定。

------------------------------------------------------------
五、原记忆扫描写法建议
------------------------------------------------------------

标题：结束学习后准备与小尤去吃饭
正文：
用户与小尤在资料室学习了一段时间。学习结束后，小尤跟随用户走出资料室，来到资料室外的走廊。她精神上有轻微疲倦，但整体轻松，准备和用户一起去吃饭。

备注：
小尤当前所在位置：资料室外的走廊。
携带装备：平板电脑、中性笔、学习资料。
mp: 85/100
hp: 100/100
当前目标：准备和用户一起去吃饭。

这样扫描时更容易识别地点、HP、MP、装备和当前目标。
