推荐状态块格式

        
初始化资料和动态变化分开看。扩展字段只保留入口，具体字段仍回到“字段设置”里管理。

        

          

            
              初始化资料
              写在角色卡 / 世界书 / 预设里，用来建立初始状态。
            
            

              
块名
status_panel

              
用途
创建新角色、补空字段，不覆盖游玩后的实时状态。

            

            
```status_panel
type: character
id: lin_pianpian
name: 林翩翩
aliases: 翩翩, 小林
group: 主角团
alive_status: 存活
hp: 82/100
mp: 34/60
location: 雪巷客栈
relationship: 长期同行
effects: 旧伤, 疲惫
summary: 雪巷捡回的少女，目前与玩家长期同行。
```

          


          

            
              动态变化
              写在 AI 回复末尾，用来记录本轮变化。
            
            

              
块名
status_panel_update

              
用途
更新当前状态，优先级高于初始化资料。

            

            
```status_panel_update
type: character
id: lin_pianpian
name: 林翩翩
location: 雪巷客栈二楼
effects_add: 失血
summary: 本轮冲突后旧伤加重，体力明显下降。
```

          

        


        

          

            扩展字段
            扩展字段只保留写法入口，显示名称和排序统一去“字段设置”里管理。
          

          extra.腿部状态: 左腿轻微扭伤
