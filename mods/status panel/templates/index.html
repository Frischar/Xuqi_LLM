Status Panel Mod

角色状态面板

扫描聊天、世界书、记忆、预设和当前角色卡里的状态块，提取成玩家能看的状态栏。聊天里的动态状态块会尽量从正文中纯隐藏，调试信息显示在状态栏面板内。

回到聊天

聊天页表格列

这里决定聊天页状态面板第一层显示哪些列。扩展字段填入角色资料后，可以在这里把它提升为表格列，不用手改 field_schema.json。默认情况下，展开详情仍会显示全部扩展字段；如果想避免重复，可在上方勾选“详情隐藏表格扩展字段”。

常用固定字段：name、alive_status、hp、mp、location、status_effects。 扩展字段写法：extra.腿部状态、extra.精神状态、extra.装备。

推荐状态块格式

      
建议放到角色卡、世界书或记忆里。预设则用来要求模型在发生状态变化时输出 status_panel_update。动态更新块建议只写本轮发生变化的字段。

      
```status_panel
type: character
id: lin_pianpian
name: 林翩翩
group: 主角团
alive_status: 存活
hp: 82/100
mp: 34/60
location: 雪巷客栈
relationship: 长期同行
effects: 旧伤, 疲惫
summary: 雪巷捡回的少女，目前与玩家长期同行。
```

```status_panel_update
type: character
id: lin_pianpian
name: 林翩翩
location: 雪巷客栈二楼
effects_add: 失血
summary: 本轮冲突后旧伤加重，体力明显下降。
```

扩展字段示例：
extra.腿部状态: 左腿轻微扭伤
extra.精神状态: 稳定
extra.装备: 平板电脑、画笔

说明：HP / MP / 生死状态属于高风险字段，默认会进入待确认；位置、关系、状态增删、摘要和 extra.* 扩展字段会自动应用。表格列可在上方“聊天页表格列”里直接配置；进阶作者也可以继续改 data/field_schema.json。
