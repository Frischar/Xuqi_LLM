# Xuqi LLM Chat Android 版详细说明书

版本：1.1.4

适用范围：`E:\AI chat 项目\xuqi_llm聊天_android` 当前 Android 包所包含的移动端功能。

## 1. 产品定位

Android 版是一个以本地存档为中心的角色聊天应用，核心由以下几部分组成：

- 聊天：与当前角色进行对话。
- 设置：管理模型、头像、背景、音乐、检索、立绘等运行配置。
- 预设：管理系统提示、写作规则模块、额外规则块。
- 角色卡：编辑当前角色卡、人设、多角色信息、剧情阶段。
- 创意工坊：按游戏阶段触发固定动作，目前支持“播放音乐”或“弹出图片”。
- 记忆库：保存长期记忆，参与后续对话。
- 世界书：按关键词命中后，将相关设定注入当前轮对话。

应用所有数据默认保存在手机本地存档中，不依赖云端同步。

## 2. 页面结构总览

侧边栏包含 7 个主页面：

1. 聊天
2. 设置
3. 预设
4. 角色卡
5. 创意工坊
6. 记忆库
7. 世界书

侧边栏同时提供：

- 当前存档选择
- 存档重命名
- 重置当前存档
- 切换主题
- 导出全部数据

## 3. 存档机制

应用内置多个本地存档槽位，每个存档都会独立保存：

- 模型配置关联的当前运行状态
- 当前角色卡
- 当前 Persona 摘要
- 当前聊天记录
- 记忆库
- 世界书
- 当前预设库
- 用户资料与头像
- `temp` 阶段计数

### 3.1 `temp` 的规则

`temp` 是创意工坊阶段计算的唯一依据。

- `temp` 绑定当前存档里的当前角色卡
- 每次点击“结束当前对话”，`temp + 1`
- 修改当前角色卡内容时，`temp` 清零
- 导入新的角色卡时，`temp` 清零
- 应用单人/多人模板时，`temp` 清零
- 切换到新存档时，读取该存档自己的 `temp`

### 3.2 当前阶段判定

当前版本固定规则：

- A 阶段：`temp = 0 ~ 2`
- B 阶段：`temp = 3 ~ 5`
- C 阶段：`temp >= 6`

这个阶段会显示在：

- 侧边栏存档摘要
- 设置页“当前游戏阶段”
- 创意工坊页“阶段判定”

## 4. 聊天页说明

聊天页是主交互页面，包含：

- 当前 Persona 名称
- 当前 Persona 开场摘要
- 消息列表
- 输入框
- 发送按钮
- 结束当前对话
- 导出聊天记录

### 4.1 发送消息时会发生什么

每轮发送时，系统会组合以下内容发给模型：

1. 当前 Persona 的系统设定
2. 当前激活预设生成的提示词
3. 用户资料摘要
4. 当前记忆库中的全部长期记忆 recap
5. 已删除记忆的阻断提示
6. 本轮命中的世界书条目
7. 最近若干轮聊天历史
8. 当前用户输入

### 4.2 结束当前对话

点击“结束当前对话”后，会：

1. `temp + 1`
2. 尝试将当前聊天整理为一条新的长期记忆
3. 写入记忆库，或在重复时跳过
4. 清空当前聊天消息列表
5. 刷新当前路由界面
6. 重新计算创意工坊阶段状态

### 4.3 聊天导出

“导出记录”会把当前存档聊天导出为文本文件。

导出内容包含：

- 时间戳
- 发言角色
- 文本正文

## 5. 设置页说明

设置页分为多个配置模块。

### 5.1 模型配置

可配置项：

- 接口预设
- API URL
- API Key
- 模型名
- Temperature
- 超时秒数
- 上下文保留轮数
- 最大输出 Token
- 主题

辅助功能：

- 测试连接：向模型发起一次最小测试请求
- 保存并刷新模型配置：保存后立即刷新界面与运行状态

### 5.2 界面与背景

可配置项：

- 界面透明度
- 背景遮罩强度
- 背景图 URL
- 上传背景图
- 清空背景

说明：

- 背景图支持远程地址和本地上传
- 背景遮罩用于提高文字可读性
- 透明度仅影响内容层，不会让侧边栏变成透明抽屉

### 5.3 检索与重排

用于扩展记忆/知识检索能力的配置入口。

可配置项：

- 嵌入 API URL
- 嵌入 API Key
- 嵌入模型名
- 嵌入字段选择：标题 / 正文 / 标签 / 备注
- 召回条数 TopK
- 是否启用重排序
- 重排 API URL
- 重排 API Key
- 重排模型名
- 重排保留条数 TopN

当前说明：

- 这是 Android 版中已经复刻好的配置入口
- 主要用于与外部 embedding / rerank 服务配套
- 具体是否启用取决于你的后端或本地服务接入方式

### 5.4 表情立绘

可配置项：

- 是否启用立绘路径配置
- 立绘目录

当前版本用途：

- 先提供配置入口
- 头像回退逻辑可读取立绘目录中的 `avatar.png`
- 后续如继续扩展，可直接把表情立绘接进聊天界面

### 5.5 头像资料

包含：

- 用户头像预览
- 角色头像预览
- 用户显示名
- 用户昵称
- 用户设定
- 备注
- 用户头像地址
- 用户头像上传
- 角色头像地址
- 角色头像上传

头像逻辑：

- 用户头像优先使用用户头像配置
- 角色头像优先使用角色头像配置
- 如果角色头像留空，系统会尝试回退到立绘目录中的 `avatar.png`

上传逻辑：

- Android 端会通过系统文件选择器选择图片
- 成功后会转为本地可用数据并立即刷新预览

### 5.6 当前游戏阶段

这里会显示：

- 当前阶段是 A / B / C 中哪一档
- 阶段由 `temp` 自动计算

并提供“打开创意工坊”快捷入口。

### 5.7 当前人设摘要

这里是当前 Persona 的快速编辑入口，可直接修改：

- 角色名
- 开场白
- 系统设定

适合做轻量调试，但不等于完整角色卡编辑。

### 5.8 剧情音乐

提供一个独立于创意工坊的手动音乐控制区。

可配置项：

- 音乐预设
- 自定义音乐 URL

可用操作：

- 播放
- 暂停
- 停止

用途：

- 手动测试 BGM
- 在不依赖创意工坊的情况下直接试听音乐

### 5.9 导入与导出

支持：

- 导出全部数据
- 导入全部数据

说明：

- 导出的数据是完整本地状态 JSON
- 导入后会直接替换当前手机上的本地状态
- 导入完成后会重建页面状态、存档状态、角色卡、预设等内容

## 6. 预设页说明

预设用于控制“系统规则层”，主要影响模型输出方式，而不是替代角色卡。

### 6.1 预设库

每个存档都有自己独立的预设库。

支持操作：

- 新增预设
- 复制预设
- 设为当前
- 删除预设
- 导入预设
- 导出当前预设

### 6.2 当前预设编辑

每个预设包含：

- 预设名称
- 启用状态
- 基础系统提示
- 常用模块
- 额外规则块

### 6.3 常用模块

当前内置模块包括：

- 防抢话
- 短段落
- 长段落
- 第二人称
- 第三人称
- 抗重复
- 避免强收尾

互斥关系：

- 短段落 / 长段落 互斥
- 第二人称 / 第三人称 互斥

### 6.4 额外规则块

每个规则块包含：

- 标题
- 启用状态
- 正文

支持：

- 新增规则块
- 删除规则块
- 全部折叠
- 全部展开
- 搜索规则块

### 6.5 预设实际生效方式

当前激活预设会在每次发送消息时被拼入系统提示层。

拼接顺序大致为：

1. 基础系统提示
2. 已启用的模块规则
3. 已启用的额外规则块

## 7. 角色卡页说明

角色卡页用于管理当前存档绑定的人设卡。

### 7.1 基础字段

支持编辑：

- `name`
- `tags`
- `description`
- `personality`
- `scenario`
- `first_mes`
- `mes_example`
- `creator_notes`

### 7.2 剧情阶段 `plotStages`

支持动态增删与编辑。

每个阶段包含：

- 阶段说明
- 阶段规则

### 7.3 人设列表 `personas`

支持多人设结构。

每个 persona 可编辑：

- 名称
- 描述
- 性格
- 场景
- 备注

### 7.4 模板

支持一键应用：

- 单人模板
- 多人模板

模板用途：

- 快速生成一套可编辑的基础角色卡结构

### 7.5 导入导出

支持：

- 导入 JSON / 文本角色卡
- 导出当前角色卡

导入角色卡后会：

- 替换当前存档角色卡
- 重算当前 Persona 摘要
- 清零 `temp`
- 重新同步创意工坊运行状态

## 8. 创意工坊说明

创意工坊是 Android 版的“阶段动作系统”。

它不开放脚本，不执行任意逻辑，只做固定预设动作。

当前固定支持两种动作：

1. 播放音乐
2. 弹出图片

### 8.1 触发原则

创意工坊按当前存档角色卡内的规则运行。

触发依据只有两项：

- 当前创意工坊是否启用
- 当前 `temp` 所在阶段 A / B / C

### 8.2 阶段规则

页面中会展示规则列表。

每条规则都包含：

- 标题
- 是否启用
- 触发阶段
- 动作类型
- 备注

并且动作类型只能二选一：

- 播放音乐
- 弹出图片

切换动作类型时：

- 另一组动作字段会被清空
- 页面上只保留当前动作对应的配置面板

### 8.3 音乐动作

音乐动作可配置：

- 音乐预设
- 音乐 URL
- 上传音乐
- 自动播放
- 循环播放
- 音量

触发后效果：

- 会切换底部 `bgmPlayer`
- 可自动播放
- 可循环

### 8.4 图片动作

图片动作可配置：

- 弹窗标题
- 图片 URL
- 上传图片
- 图片说明

触发后效果：

- 打开独立图片弹窗
- 显示触发阶段
- 显示标题、图片与说明

### 8.5 页面工具

创意工坊页支持：

- 启用/禁用创意工坊
- 新增规则
- 搜索规则
- 全部折叠
- 全部展开

### 8.6 当前版本的重要限制

- 规则动作只能是“图片”或“音乐”之一
- 当前阶段固定为 A / B / C 三段
- 阶段阈值当前写死为 `0-2 / 3-5 / 6+`
- 不支持任意脚本
- 不支持复杂条件表达式

### 8.7 什么时候会触发

当前版本主要在以下场景同步创意工坊状态：

- 应用启动并渲染当前存档
- 载入角色卡
- 应用角色模板
- 结束当前对话导致 `temp` 变化
- 手动切换页面/刷新当前运行态

## 9. 记忆库说明

记忆库用于长期存储“对话总结后的结果”。

### 9.1 记忆的来源

来源有两种：

1. 用户手动新增
2. 点击“结束当前对话”后自动总结生成

### 9.2 自动总结逻辑

结束当前对话时，系统会优先尝试让模型返回一个记忆 JSON，字段为：

- `title`
- `content`
- `tags`
- `notes`

如果模型生成失败，会使用本地回退逻辑生成一条记忆。

### 9.3 长对话处理

如果当前对话太长：

- 系统会先把长对话切成多段
- 逐段压缩为摘要
- 再基于摘要生成最终记忆

这样可以降低长对话直接总结失败的概率。

### 9.4 删除记忆的保护机制

删除记忆后，不只是从界面里消失。

系统还会：

- 把它加入“已删除记忆”列表
- 在后续自动总结时显式提醒模型“不要把这些旧记忆写回”

这能减少“删除了又被总结重新写回来”的情况。

### 9.5 记忆页可用功能

- 新增记忆
- 编辑标题、正文、标签、备注
- 搜索记忆
- 全部折叠
- 全部展开
- 删除记忆

### 9.6 记忆如何参与对话

每轮发送消息时，当前记忆库会被整理成长期前情提要，作为系统提示的一部分拼入上下文。

## 10. 世界书说明

世界书用于“按输入命中关键词，再把设定注入到当前轮对话”。

### 10.1 世界书设置

可配置：

- 是否启用世界书
- 每轮最多注入条数
- 默认命中模式：任意命中 / 全部命中
- 是否忽略大小写
- 是否完整词匹配

### 10.2 词条字段

每条世界书词条可配置：

- 标题
- 主触发词
- 次触发词
- 正文内容
- 优先级
- 命中模式
- 是否启用
- 忽略大小写
- 完整词匹配
- 备注

### 10.3 命中逻辑

每轮发送消息时，会对用户本轮输入执行匹配：

- 先筛选启用词条
- 按命中模式判断是否命中
- 按优先级排序
- 截取最多注入条数
- 把命中条目作为系统设定注入这一轮请求

### 10.4 世界书页可用功能

- 新增词条
- 搜索词条
- 全部折叠
- 全部展开
- 编辑词条
- 删除词条

## 11. 头像、消息气泡与显示逻辑

聊天消息中的头像逻辑如下：

- 用户消息优先使用用户头像
- 角色消息优先使用角色头像
- 角色头像为空时，尝试使用立绘目录下头像
- 头像与聊天气泡顶部对齐显示

聊天页顶部的 Persona 名称逻辑如下：

- 优先显示当前推导出的真实角色名
- 如果顶层角色卡名称还是模板默认名，会优先使用 persona 列表中的角色名

## 13. 推荐使用流程

### 方案 A：快速开始

1. 进入“设置”填好模型配置
2. 点击“测试连接”
3. 进入“角色卡”应用单人模板
4. 在“设置”里补充头像与用户资料
5. 在“预设”中打开你需要的写作模块
6. 回到“聊天”开始对话

### 方案 B：带剧情阶段的玩法

1. 配置角色卡
2. 进入“创意工坊”为 A/B/C 阶段设置音乐或图片
3. 进入“世界书”补充世界观关键词
4. 每聊完一段点击“结束当前对话”
5. 让系统累积 `temp` 和长期记忆
6. 随着阶段推进，创意工坊动作会变化

### 方案 C：长期陪伴式玩法

1. 配置用户资料
2. 让记忆库持续积累
3. 通过预设控制说话风格
4. 用世界书维持固定背景设定
5. 定期导出全部数据做备份

## 14. 常见问题

### 14.1 为什么我修改角色卡后阶段变回 A？

因为 `temp` 会在修改、导入或套用新角色卡时清零，这是当前版本的固定规则。

### 14.2 为什么删除记忆后它没有马上回来？

这是正常的。系统会把已删除记忆加入阻断列表，防止自动总结时又重新写回。

### 14.3 为什么创意工坊只会执行音乐或图片中的一种？

这是当前设计要求。每条规则固定只能有一个动作类型，目的是保持行为可控。

### 14.4 预设和角色卡有什么区别？

- 角色卡：定义“角色是谁”
- 预设：定义“模型怎么说”

### 14.5 世界书和记忆有什么区别？

- 世界书：按关键词触发的结构化设定
- 记忆：从历史对话中沉淀出来的长期前情

## 15. 数据安全与维护建议

- API Key 仅保存在本地设备
- 长期使用前，建议先导出一份全部数据备份
- 进行大规模角色卡改动前，建议先导出当前角色卡和预设
- 如果要做多人设/多剧情实验，建议分不同存档槽位进行
